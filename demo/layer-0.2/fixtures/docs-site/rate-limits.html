Rate-Limits und Quoten

DMH SME Demo Cloud setzt Rate-Limits durch, um die Stabilität der Plattform für alle Kunden zu sichern. Diese Seite beschreibt die Limits pro Tarif, das Verhalten beim Überschreiten und wie Sie eine Quoten-Erhöhung beantragen können.

Die folgenden Werte gelten pro Kunden-Account und beziehen sich auf die API-Schnittstelle api.dmh-demo.example. Das Burst-Limit erlaubt kurzfristig die doppelte Spitze, ohne dass sofort 429 zurückkommt.

Limits pro Endpunkt

Zusätzlich zu den Tarif-Limits gelten Limits pro Endpunkt. Diese schützen vor versehentlichen oder bösartigen Lastspitzen auf besonders teuren Endpunkten und greifen unabhängig vom Tarif-Limit.

  • POST /api/orders — zusätzlich 30 Anfragen pro Minute pro Kunden-Account (Anti-Abuse-Schutz vor automatisierten Bestell-Bot-Wellen)
  • POST /api/auth/login — 10 Anfragen pro Minute pro IP, danach 5-minütiger Cool-Down. Schutz vor Credential-Stuffing-Angriffen.
  • POST /api/refund — 5 Anfragen pro Minute pro Kunden-Account; ausnahmsweise Erhöhung nur via Support-Workflow möglich.

Verhalten beim Überschreiten

Wird das Limit überschritten, antwortet der Server mit HTTP 429 Too Many Requests und dem Header Retry-After: <Sekunden>. Wir empfehlen einen Client-Side Exponential-Backoff beginnend bei 1 Sekunde, Verdoppelung pro Fehlversuch, mit Jitter ±20 %. Geben Sie nach drei Fehlversuchen auf und melden den Vorfall an Ihr eigenes Monitoring — das deutet meist auf einen anhaltenden Engpass auf unserer Seite hin, der manuell untersucht werden muss.

Quoten-Erhöhung

Eine vorübergehende Quoten-Erhöhung (z. B. für eine Black-Friday-Kampagne) kann über den Support-Workflow support@dmh-demo.example mindestens 5 Werktage im Voraus beantragt werden. Permanente Erhöhungen erfordern einen Tarif-Wechsel und werden vom Account-Management bearbeitet. Vorübergehende Erhöhungen sind auf maximal 48 Stunden begrenzt und werden danach automatisch zurückgesetzt.
