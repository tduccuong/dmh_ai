Webhooks — Outbound-Events konfigurieren

Webhooks sind der primäre Mechanismus, mit dem DMH SME Demo Cloud Ihrem System Ereignisse signalisiert. Diese Seite beschreibt die unterstützten Events, das Registrieren eines Endpunkts, Signatur-Verifikation und das Retry-Verhalten bei Fehlern.

Unterstützte Events

Die folgenden Event-Typen werden zum aktuellen Zeitpunkt von DMH SME Demo Cloud ausgeliefert. Jedes Event hat ein stabiles JSON-Schema, das im Reiter Schema-Vorschau im Adminportal einsehbar ist.

  • order.created — neue Bestellung wurde angelegt
  • order.cancelled — Bestellung wurde storniert
  • invoice.paid — Rechnung wurde bezahlt
  • user.signup — neuer Benutzer hat sich registriert
  • user.role_changed — Rolle wurde geändert

Endpunkt registrieren

  1. Im DMH-Adminportal unter /admin/webhooks → Neuer Endpunkt die Ziel-URL eingeben. Erforderlich ist HTTPS — HTTP-URLs werden mit einer 400-Antwort abgelehnt. Selbst-signierte Zertifikate akzeptieren wir nicht; bitte verwenden Sie ein gültiges öffentliches Zertifikat (z. B. von Let's Encrypt).
  2. Ein Webhook-Secret wird automatisch generiert und einmalig angezeigt. Mit diesem Secret werden alle ausgehenden Requests signiert (HTTP-Header X-DMH-Signature, HMAC-SHA256 über den unveränderten Request-Body).
  3. Mindestens ein Event in der Auswahlliste anhaken. Mehrfachauswahl ist möglich; pro Endpunkt empfehlen wir maximal 5 Events, um die Endpunkt-Verantwortung übersichtlich zu halten.

Retry-Verhalten

Schlägt ein Webhook-Zustellversuch fehl (HTTP-Status >= 500 oder Timeout > 10 Sekunden), wird die Zustellung bis zu sechs mal mit exponentiellem Backoff wiederholt. Die genauen Intervalle sind: 30 Sekunden, 2 Minuten, 10 Minuten, 1 Stunde, 6 Stunden, 24 Stunden. Nach dem letzten Fehlversuch wird der Endpunkt automatisch deaktiviert und eine Benachrichtigung an die im Admin-Profil hinterlegte E-Mail verschickt.

Signatur prüfen

Beispiel-Code in Python zur Verifikation:

import hmac, hashlib
def verify(body: bytes, sig_header: str, secret: str) -> bool:
    expected = hmac.new(secret.encode(), body, hashlib.sha256).hexdigest()
    return hmac.compare_digest(expected, sig_header)

Verwenden Sie ausdrücklich hmac.compare_digest und nicht ==, um Timing-Angriffe zu vermeiden. Wenn die Verifikation in einem anderen Programmiersprachen-Stack erfolgt, finden Sie im Adminportal unter /admin/webhooks/snippets äquivalente Beispiele für Node.js, Go, Ruby und PHP.
