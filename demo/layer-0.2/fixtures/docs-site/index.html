DMH SME Demo Cloud — Produktdokumentation

Diese Dokumentation gilt für die DMH SME Demo Cloud, Version 4.7.0, Stand Q2/2026. Sie richtet sich an Administrator:innen und Entwickler:innen, die DMH SME Demo Cloud in einem mittelständischen Unternehmen einführen oder betreiben.

Was ist die DMH SME Demo Cloud?

DMH SME Demo Cloud ist eine fiktive Demo-Instanz, an der wir die Integrationsmuster zeigen: Single Sign-On, eingehende und ausgehende Webhooks, Quotas und Rate-Limits, sowie das Tarif-Modell. Alle in dieser Dokumentation genannten Namen, Endpunkte und Zahlenwerte sind synthetisch und dienen ausschließlich der Demonstration der Indexierungs-Pipeline.

Übersicht der Module

Die folgenden Module sind Teil der Standard-Installation. Jedes Modul hat eine eigene Konfigurationsseite und ist über die linke Navigationsleiste im Adminportal erreichbar.

Themen in dieser Dokumentation

Lesen Sie die folgenden Seiten in der angegebenen Reihenfolge, wenn Sie eine Erstinstallation durchführen. Wenn Sie bereits eine laufende Instanz haben, springen Sie direkt zu dem Thema, das Sie interessiert.

  • Single Sign-On mit Azure AD, Google Workspace und Okta — Identity-Provider anbinden, JIT-Provisioning, SAML-Fehlerdiagnose
  • Webhooks — Outbound-Events konfigurieren — Endpunkte registrieren, Signatur-Verifikation, Retry-Verhalten
  • Rate-Limits und Quoten — Anfragen pro Minute pro Tarif, 429-Verhalten, Quoten-Erhöhungen beantragen

Support und Eskalation

Bei Problemen mit der DMH SME Demo Cloud erstellen Sie ein Ticket im Kundenportal unter support.dmh-demo.example. Eine direkte Mail an support@dmh-demo.example wird automatisch in das Ticketsystem übernommen. Für kritische Produktionsstörungen außerhalb der Geschäftszeiten steht Premium-Kunden zusätzlich eine 24/7-Notfall-Hotline zur Verfügung, deren Nummer im Premium-Vertrag hinterlegt ist. Antwortzeiten richten sich nach Ihrem Tarif: Starter 24 Stunden, Business 4 Stunden, Enterprise 1 Stunde.

Versionshinweise

Mit Version 4.7.0 wurde das SCIM-Provisioning für Microsoft Entra ID stabilisiert und das Webhook-Retry-Backoff von linearer auf exponentielle Form umgestellt. Eine vollständige Liste finden Sie im Changelog unter /admin/changelog im Adminportal.
